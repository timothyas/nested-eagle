\vspace{-2ex}
\section{Goal}

\vspace{-2.5ex}
Develop a global medium range weather prediction model that:
\vspace{.5ex}
\begin{itemize}
    \item captures synoptic scale dynamics
    \item represents precipitation at scale useful for decision makers
    \item produces forecasts at a low computational expense
\end{itemize}

\vspace{-1.5ex}
\section{Data}

\vspace{-2.5ex}
\begin{itemize}
    \item Train on GFS \& HRRR ``Analysis'' (fhr=0) for all\\variables, except
        precipitation, which uses 0-6h\\forecast accumulations
        \vspace{.25ex}
    \item Implement nested or ``stretched'' grid approach,\\following Met
        Norway [1]:
        cut out CONUS portion from GFS, stick in HRRR
        \vspace{.25ex}
    \item Conservatively regrid archived GFS: 1/4$^\circ$$\rightarrow$1$^\circ$ and
        HRRR: 3km$\rightarrow$15km
        \vspace{.25ex}
    \item Use full archives available on NCAR RDA and AWS:
        \begin{itemize}
            \item Training: Feb 2015-Jan 2023
            \item Validation: Feb 2023-Jan 2024
            \item Testing Feb 2024-Jan 2025
        \end{itemize}
        \vspace{.25ex}
    \item Use \texttt{ufs2arco} [2] for all data processing
\end{itemize}

\vspace{-1.5ex}
\section{Design Choices that Mattered}

\vspace{-2.5ex}
\begin{itemize}
    \item Moving from ``GraphCast Style'' multimesh [3] to\\single, high resolution
        mesh with shifted window\\processor removed GFS/HRRR boundary artifacts
        \vspace{.25ex}
    \item Reducing CONUS loss weight 50\%$\rightarrow$10\% improved skill
        significantly
        \vspace{.25ex}
\end{itemize}
